## 2.7 系统工具

### 2.7.1 任务管理器

![适中](../../图片/medium.svg)

任务管理器是Windows系统自带的系统监控工具，用于查看和管理当前运行的程序、进程以及系统资源使用情况。

![任务管理器总览](../../图片/xt-xtgj-rwglq-0.png)

以上是任务管理器常见功能区的总览。对于一般用户，重点认识`进程`、`性能`和`启动应用`三大部分即可。

#### 2.7.1.1 打开任务管理器

有以下几种常用方式：
- 快捷键 `Ctrl`+`Shift`+`Esc` 直接打开
- 快捷键 `Ctrl`+`Alt`+`Del` 后选择"任务管理器"
- 右键点击任务栏空白处，选择"任务管理器"
- 右键点击开始菜单，选择"任务管理器"

#### 2.7.1.2 查看运行中的程序

![任务管理器进程页](../../图片/xt-xtgj-rwglq-1.jpg)

任务管理器通常会显示"进程"选项卡（也可能停留在你上次查看的页面），其中列出了当前正在运行的应用程序和后台进程，以及它们占用的CPU、内存、磁盘、网络资源。

>[!TIP]
> 在Windows 10下，如果任务管理器界面过于简化，点击左下角的"详细信息"按钮可展开完整视图。

#### 2.7.1.3 结束任务

当某个程序无响应，或你需要手动结束某个进程时：
1. 在任务管理器中找到该程序
2. 选中后点击右下角（Windows 11在右上角）的"结束任务"按钮
3. 或直接右键选择"结束任务"

>[!CAUTION]
> 结束任务会导致该程序未保存的数据丢失。请优先尝试正常关闭程序。

#### 2.7.1.4 查看系统性能

切换到"性能"选项卡，可以查看CPU、内存、磁盘、网络、GPU等硬件的实时运行状态。

##### CPU

![任务管理器CPU](../../图片/xt-xtgj-rwglq-cpu.png)

**型号信息**：如"12th Gen Intel Core i9-12900HX"表示第12代英特尔酷睿i9处理器，12900HX是具体型号。

**关键指标**：
- **利用率**：CPU当前被占用的百分比，100%表示满载
- **速度**：当前运行频率（如2.38 GHz），基准速度是参考频率（如2.30 GHz）。**购买时常见的宣传频率多为最高加速频率，通常是在特定条件下由少数核心达到的峰值频率，不等于所有核心长期稳定运行频率。**
- **进程**：当前运行的程序数量（并不等于你打开的软件数量，甚至于远多，因为一个软件往往有多个进程，而且还有很多后台进程。）
- **线程**：程序执行的最小单元，数量越多并行处理能力越强
- **句柄**：系统资源的引用标识
- **正常运行时间**：电脑自开机以来的持续运行时间
- **内核**：物理核心数，如16核
- **逻辑处理器**：包含超线程后的总线程数，如24个
- **虚拟化**：是否启用了硬件虚拟化技术
- **L1/L2/L3缓存**：CPU内置的高速缓存，L1通常最快且最小，L3通常更大但延迟更高；缓存大小会影响性能，但需结合架构和实际负载综合判断。

##### 内存

![任务管理器内存](../../图片/xt-xtgj-rwglq-memory.png)

**容量信息**：如"32.0 GB"表示总物理内存容量。

**关键指标**：
- **使用中**：当前被占用的内存容量，括号内为压缩内存量
- **可用**：剩余可用内存
- **已提交**：系统承诺分配（但不一定实际使用）给程序的内存总量/上限（包括物理内存+虚拟内存）
- **已缓存**：用于缓存近期数据和文件的内存（数据通常来自磁盘-虚拟内存）
- **速度**：内存运行频率（如4800 MT/秒），频率越高通常带来更高带宽；实际体验还会受时序、延迟和平台配置影响。
- **插槽**：已使用的内存插槽数/总插槽数（如2/2表示两个插槽都已使用）
- **分页缓冲池/非分页缓冲池**：系统内核使用的内存区域，分页表示可以被移动到磁盘上，非分页表示不可以。

##### 磁盘

![任务管理器磁盘](../../图片/xt-xtgj-rwglq-disk.png)

**设备信息**：如"磁盘 0 (C: D:)"表示第一块硬盘，包含C盘和D盘两个分区。

**关键指标**：
- **活动时间**：磁盘处于活动状态的百分比，100%表示磁盘满载
- **平均响应时间**：磁盘响应请求的平均时间，越低越好
- **读取速度/写入速度**：当前数据传输速率
- **容量**：硬盘总容量
- **类型**：如"SSD (NVMe)"表示固态硬盘，采用NVMe协议，速度比传统机械硬盘(HDD)快得多

##### 网络

![任务管理器网络](../../图片/xt-xtgj-rwglq-network.png)

**适配器信息**：如"Intel Wi-Fi 6 AX201"表示英特尔Wi-Fi 6无线网卡。

**关键指标**：
- **发送/接收**：当前上传和下载的实时速度（默认为bps，注意8 bps = 1 B/s）
- **SSID**：连接的Wi-Fi网络名称
- **连接类型**：无线标准，如802.11ax即Wi-Fi 6，为IEEE标准号
- **IPv4地址/IPv6地址**：设备在网络中的地址（本地地址）
- **信号强度**：Wi-Fi信号质量，满格表示信号良好

##### GPU

![任务管理器GPU](../../图片/xt-xtgj-rwglq-gpu.png)

**型号信息**：如"NVIDIA GeForce RTX 4070 Laptop GPU"表示英伟达RTX 4070笔记本显卡。

**关键指标**：
- **利用率**：GPU当前负载百分比，分为3D（3D性能，打游戏耗的是这个）、Copy（简单理解为显存带宽占用）、Video Encode（视频编码）、Video Decode（视频解码）等不同类型
- **专用GPU内存**：显卡自带的显存（如8GB），仅供显卡使用，**重点看这个**
- **共享GPU内存**：借用系统内存作为显存使用，性能很弱（如15.9GB）
- **GPU内存**：专用+共享的总和
- **温度**：显卡当前温度，过高可能导致降频
- **驱动程序版本**：显卡驱动版本号，影响性能和兼容性

#### 2.7.1.5 管理启动项

切换到"启动应用"选项卡，可以查看开机时自动启动的程序：
- 右键点击可禁用或启用启动项
- 禁用不必要的启动项可加快开机速度

>[!TIP]
> 部分软件的自启动并不会在该选项卡中展示出来。这是由不同的自启动策略导致的，如果想要关闭自启动的软件不在该列表里，建议到对应软件的设置项当中进行设置。

### 2.7.2 命令提示符

![适中](../../图片/medium.svg)

命令提示符是Windows系统自带的命令行工具，用于通过文本命令与操作系统进行交互。它可以帮助你快速执行各种任务，如文件管理、系统信息查看、网络诊断等。对于电脑初学者来说，学习这些基本命令能帮你解决日常使用中遇到的常见问题，让电脑管理变得更高效。

#### 2.7.2.1 打开命令提示符

有以下几种常用方式：
- 按 `Win` + `R` 键，输入 `cmd` 并回车
- 在开始菜单搜索"命令提示符"或"cmd"
- 在文件资源管理器的地址栏输入 `cmd` 并回车，可在当前目录打开
- 右键开始菜单，选择"命令提示符"或"Windows PowerShell"（两者类似，但PowerShell更强大）

![命令提示符窗口（来源于网络）](../../图片/xt-cmd-window.jpg)

<center><small>命令提示符窗口（来源于网络）</small></center>

>[!TIP]
> 如果需要以管理员身份运行，右键选择"以管理员身份运行"，这可以执行需要更高权限的命令。有些命令如系统修复需要管理员权限。
> 
> 另外，按向上箭头键（↑）或向下箭头键（↓）可以快速调用之前输入过的命令，节省重复输入的时间。

#### 2.7.2.2 基本导航命令

命令提示符默认打开在用户文件夹，你可以使用命令在目录间切换。

**查看当前目录**：
```
cd
```
显示当前所在路径。

**切换目录**：
```
cd C:\Users
```
切换到C盘的Users文件夹。

```
cd ..
```
返回上级目录。

```
cd \
```
切换到根目录（C盘根目录）。

**切换驱动器**：
```
D:
```
切换到D盘。注意，驱动器字母后要加冒号。

**列出文件和文件夹**：
```
dir
```
列出当前目录下的所有文件和文件夹，包括大小、修改时间等信息。

```
dir /w
```
以宽格式列出文件名，方便快速浏览。

```
dir /ad
```
只列出文件夹（目录）。

```
dir *.txt
```
只列出.txt格式的文件。

>[!TIP]
> 使用Tab键可以自动补全文件名或路径，节省输入时间。比如输入 `cd Des` 然后Tab，会补全为 `cd Desktop`。

##### 2.7.2.2.1 获取帮助

**查看所有可用命令**：
```
help
```
列出所有内置命令及其简要说明。

**查看特定命令的帮助**：
```
help 命令名
```
如 `help dir` 会显示dir命令的详细用法。

>[!TIP]
> 如果忘记某个命令的用法，help命令是最好的老师。它会告诉你命令的语法和参数。

#### 2.7.2.3 文件和文件夹操作

![命令提示符文件操作（来源于网络）](../../图片/xt-cmd-dir.JPG) <!-- 显示dir命令的输出结果，列出当前目录的文件和文件夹列表。资源位置：运行cmd，输入dir并截图 -->

<center><small>命令提示符文件操作（来源于网络）</small></center>

这些操作可以帮你管理文件，比如整理资料、备份重要文件等。

**创建文件夹**：
```
mkdir 新文件夹
```
或
```
md 新文件夹
```
创建一个名为"新文件夹"的文件夹。

**删除文件**：
```
del 文件名.txt
```
删除指定的文件。

**删除文件夹**：
```
rmdir 文件夹名
```
删除空的文件夹。如果文件夹不空，需要先删除里面的内容，或用 `rmdir /s 文件夹名` 删除整个文件夹及其内容（危险操作）。

**复制文件**：
```
copy 源文件.txt 目标文件.txt
```
复制文件。

**复制文件夹**：
```
xcopy 源文件夹 目标文件夹 /s
```
复制文件夹及其所有子文件夹。/s参数表示包括子文件夹。

**移动文件**：
```
move 源文件.txt C:\目标文件夹\
```
移动文件到指定位置。

**重命名文件或文件夹**：
```
ren 旧名.txt 新名.txt
```
重命名文件。

>[!CAUTION]
> 删除操作不可逆，请谨慎使用。删除文件夹时确保里面没有重要文件。建议先用 `dir` 查看内容确认。

#### 2.7.2.4 系统信息查看

![命令提示符系统信息（来源于网络）](../../图片/xt-cmd-ipconfig.png) <!-- 显示ipconfig命令的输出，列出网络配置信息。资源位置：运行cmd，输入ipconfig并截图 -->

<center><small>命令提示符系统信息（来源于网络）</small></center>

了解电脑状态是解决问题的第一步。

**查看系统信息**：
```
systeminfo
```
显示详细的系统信息，如操作系统版本、安装日期、内存大小等。

**查看网络配置**：
```
ipconfig
```
显示网络适配器的IP地址、子网掩码、网关等。

```
ipconfig /all
```
显示更详细的网络信息，包括MAC地址、DNS等。

**查看磁盘使用情况**：
```
wmic logicaldisk get size,freespace,caption
```
显示每个磁盘的大小和剩余可用空间。这个命令会列出电脑中所有磁盘分区的信息。

**查看运行中的进程**：
```
tasklist
```
列出当前运行的程序和进程。

#### 2.7.2.5 常见问题解决

![命令提示符网络诊断（来源于网络）](../../图片/xt-cmd-ping.jpeg) <!-- 显示ping www.baidu.com的输出结果，测试网络连接。资源位置：运行cmd，输入ping www.baidu.com并截图 -->
<center><small>命令提示符网络诊断（来源于网络）</small></center>

作为电脑基础教程，这里重点介绍初学者在日常使用中可能遇到的常见问题及其CMD解决方案。这些问题包括网络连接、文件损坏、系统卡顿等。

##### 网络连接问题

当你上网慢或无法上网时，可以用CMD诊断。

**检查网络连接**：
```
ping www.baidu.com
```
测试与百度的连接。如果能ping通，说明网络基本正常；如果超时，可能是DNS或路由问题。

**释放和续订IP地址**：
```
ipconfig /release
ipconfig /renew
```
先释放当前IP，再重新获取。这可以解决IP冲突或DHCP问题。

**重置网络设置**：
```
netsh winsock reset
```
重置Winsock设置，解决一些网络软件冲突问题。运行后需要重启电脑。

>[!CAUTION]
> 重置网络设置可能会暂时中断网络连接，并清除某些网络配置。运行后务必重启电脑，如果网络问题依旧，建议咨询专业人士。

**查看路由表**：
```
route print
```
显示网络路由信息，高级用户用来诊断路由问题。

>[!TIP]
> 如果网络问题持续，建议先重启路由器，或联系网络管理员。

##### 文件和磁盘问题

文件丢失或磁盘错误是常见烦恼。

**检查磁盘错误**：
```
chkdsk C:
```
检查C盘是否有坏扇区或文件系统错误。运行前确保没有程序在使用该盘。

```
chkdsk C: /f
```
自动修复发现的错误。

>[!CAUTION]
> 修复磁盘错误可能会修改文件系统，建议在运行前备份重要数据，以防数据丢失。

**修复系统文件**：
```
sfc /scannow
```
扫描并修复损坏的系统文件。需要管理员权限。

>[!CAUTION]
> 修复系统文件需要管理员权限，且扫描过程可能需要较长时间。不要在扫描过程中关闭命令提示符。

**查看文件内容**：
```
type 文件名.txt
```
显示文本文件的内容。

**查找文件**：
```
dir /s /b | find "关键词"
```
在当前目录及其子目录中搜索包含关键词的文件名。

##### 系统性能和进程问题

电脑卡顿时，可以查看和结束占用资源的程序。

**查看进程详情**：
```
tasklist /v
```
显示进程的详细信息，包括内存使用。

**结束进程**：
```
taskkill /PID 1234
```
结束指定进程号（PID）的程序。进程号可以在tasklist命令的结果中找到。

```
taskkill /IM notepad.exe
```
按程序名结束进程，如结束所有记事本。

>[!CAUTION]
> 结束进程可能会导致程序未保存的数据丢失。优先尝试正常关闭程序，避免误杀重要系统进程，否则可能导致系统不稳定。

**查看系统资源**：
```
systeminfo | find "内存"
```
快速查看内存信息。

##### 关机和重启

有时候需要定时关机或远程控制。

**立即关机**：
```
shutdown /s /t 0
```
立即关机。

**定时关机**：
```
shutdown /s /t 3600
```
1小时后关机（3600秒）。

**重启电脑**：
```
shutdown /r /t 0
```
立即重启。

**取消关机**：
```
shutdown /a
```
取消已计划的关机。

>[!CAUTION]
> 关机和重启命令会强制关闭电脑，可能导致未保存的数据丢失。使用前确保所有工作已保存。

>[!TIP]
> 关机命令在远程协助或批处理中很有用。

#### 2.7.2.6 实用案例

通过具体案例，你可以更好地理解命令的应用。

**案例1：备份重要文件**

假设你想备份桌面上的所有文档：
```
mkdir C:\Backup
copy C:\Users\用户名\Desktop\*.docx C:\Backup\
```
先创建备份文件夹，然后复制所有.docx文件。

**案例2：清理临时文件**

临时文件占用空间，可以定期清理：
```
del /q /f %temp%\*
```
删除临时文件夹中的所有文件。%temp%是Windows的临时文件夹路径，通常是C:\Users\用户名\AppData\Local\Temp。

**案例3：网络诊断完整流程**

当网络有问题时：
```
ipconfig /all
ping 192.168.1.1
tracert www.baidu.com
```
先查看本地网络配置，测试连接到路由器（网关），然后跟踪到百度的网络路径，查看数据包经过的节点。

**案例4：创建简单的批处理脚本**

创建一个名为cleanup.bat的文件，内容：
```
@echo off
echo 正在清理临时文件...
del /q /f %temp%\*
echo 清理完成！
pause
```
双击运行即可清理临时文件并显示结果。

>[!TIP]
> 命令提示符支持管道（|）和重定向（>）。管道|可以将一个命令的结果传递给下一个命令使用，比如 `tasklist | find "chrome"` 可以查找包含"chrome"的进程。重定向>可以将结果保存到文件，比如 `dir > filelist.txt` 将目录列表保存到名为filelist.txt的文件中。

通过学习这些命令，你可以解决很多日常电脑问题。命令提示符虽然界面简单，但功能强大，适合喜欢高效操作的用户。记住，谨慎使用删除和修改命令，避免误操作造成数据丢失。如果不确定某个命令的作用，可以先在网上搜索或用 `help 命令名` 查看帮助。
